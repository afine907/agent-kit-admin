## 发布策略

### 版本号规则

采用 [Semantic Versioning](https://semver.org/):

- **MAJOR** - 不兼容的 API 变更
- **MINOR** - 向下兼容的功能新增
- **PATCH** - 向下兼容的问题修正

### 发布周期

- **Alpha** - 内部测试
- **Beta** - 公开测试
- **RC** - 候选版本
- **Stable** - 正式版本

### 发布检查清单

```markdown
## Pre-release
- [ ] 所有测试通过
- [ ] 文档更新
- [ ] CHANGELOG 更新
- [ ] 版本号更新

## Release
- [ ] 创建 Git tag
- [ ] 构建 Docker 镜像
- [ ] 发布到 npm (CLI)
- [ ] 创建 GitHub Release

## Post-release
- [ ] 部署到生产环境
- [ ] 监控运行状态
- [ ] 收集用户反馈
```
