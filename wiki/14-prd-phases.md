# PRD 分阶段规格

## 概述

本文档定义了 Agent Kit Admin 的完整产品路线图，包含每个阶段的目标、功能、验收标准和成功指标。

---

## Phase 0: 技术验证 (Week 0)

### 目标
验证核心链路可行性。

### 交付物
- [ ] 手动发布一个 MCP 到 MinIO
- [ ] 手动下载并配置到 Claude Code
- [ ] 验证 MCP 在 Claude Code 中可正常调用

### 验收标准
- 完整的 发布 → 下载 → 配置 → 使用 流程跑通

### 时间
1 周

---

## Phase 1: MVP (Week 1-3)

### 目标
最小可用版本，核心流程闭环。

### 用户故事
- US-001: 发布 MCP
- US-002: 发布 Skill
- US-004: 安装到 Claude Code
- US-005: 安装到 Codex
- US-011: Web UI 浏览包
- US-012: Web UI 包详情
- US-013: 用户登录
- US-017: CLI OAuth 登录

### 功能清单

#### CLI
| 命令 | 说明 | 验收标准 |
|---|---|---|
| `akit login` | OAuth 登录 | 浏览器授权后获取 Token |
| `akit publish` | 发布包 | 成功上传到 MinIO，数据库记录 |
| `akit install` | 安装包 | 下载包，写入 Agent 配置 |
| `akit uninstall` | 卸载包 | 删除包文件，清理配置 |
| `akit list` | 已安装列表 | 显示所有已安装包 |
| `akit search` | 搜索包 | 返回匹配结果 |
| `akit info` | 包详情 | 显示包信息和安装命令 |

#### API
| 端点 | 说明 | 验收标准 |
|---|---|---|
| `POST /api/v1/auth/oauth/:provider` | OAuth 登录 | 重定向到授权页 |
| `GET /api/v1/auth/oauth/:provider/callback` | OAuth 回调 | 返回 JWT Token |
| `GET /api/v1/auth/me` | 当前用户 | 返回用户信息 |
| `GET /api/v1/packages` | 包列表 | 支持搜索和筛选 |
| `GET /api/v1/packages/:scope/:name` | 包详情 | 返回包信息 |
| `POST /api/v1/packages` | 创建包 | 创建成功 |
| `GET /api/v1/packages/:scope/:name/versions` | 版本列表 | 返回版本历史 |
| `POST /api/v1/packages/:scope/:name/versions` | 发布版本 | 上传成功 |
| `GET /api/v1/packages/:scope/:name/download` | 下载包 | 302 重定向到 MinIO 预签名 URL |

#### Web UI
| 页面 | 说明 | 验收标准 |
|---|---|---|
| 首页 | 包列表 | 搜索、筛选、排序 |
| 详情页 | 包详情 | 描述、版本、安装命令 |
| 登录页 | OAuth 登录 | 跳转授权 |
| 个人中心 | 我的包 | 显示已发布包 |

#### 部署
| 功能 | 说明 | 验收标准 |
|---|---|---|
| Docker Compose | 一键部署 | `docker compose up` 成功 |
| 初始化脚本 | 自动生成配置 | 生成 .env 和密码 |
| 健康检查 | 服务状态 | `/api/health` 返回正常 |

### Agent 支持
| Agent | 配置路径 | 验收标准 |
|---|---|---|
| Claude Code | `~/.claude/mcp.json` | 安装后可正常使用 |
| Codex | `~/.codex/config.toml` | 安装后可正常使用 |

### 明确不做
- ❌ 评分/评论
- ❌ 团队管理
- ❌ 依赖管理
- ❌ 包签名
- ❌ 多 Agent (Cursor 等)

### 验收标准
- [ ] 新用户从零到安装第一个 MCP < 3 分钟
- [ ] Docker 部署 < 5 分钟
- [ ] 核心流程 0 bug
- [ ] 内部团队 3 人使用 1 周无 blocker

### 时间
3 周

---

## Phase 2: 完善体验 (Week 4-6)

### 目标
补齐核心体验，可对外小范围测试。

### 用户故事
- US-006: 查看已安装列表 (增强)
- US-009: 搜索包 (增强)
- US-010: 查看包详情 (增强)
- US-014: 管理我的包
- US-015: 评分和评论
- US-016: 查看评分
- US-018: Token 认证 (CI/CD)

### 新增功能

#### CLI
| 命令 | 说明 | 验收标准 |
|---|---|---|
| `akit update` | 更新包 | 更新到最新版本 |
| `akit update --all` | 更新所有 | 批量更新 |
| `akit init` | 初始化项目 | 交互式创建 akit.json |
| `--token` 参数 | Token 认证 | CI/CD 可用 |
| `--agent` 参数 | 指定 Agent | 安装到指定 Agent |

#### API
| 端点 | 说明 | 验收标准 |
|---|---|---|
| `POST /api/v1/packages/:scope/:name/reviews` | 提交评分 | 1-5 星评分 |
| `GET /api/v1/packages/:scope/:name/reviews` | 评分列表 | 返回评分和评论 |
| `GET /api/v1/packages/:scope/:name/stats` | 包统计 | 下载量、评分统计 |
| `GET /api/v1/auth/api-keys` | 列出 API Key | 返回 key 列表（不含完整 key） |
| `POST /api/v1/auth/api-keys` | 生成 API Key | 返回完整 key（仅此一次） |
| `DELETE /api/v1/auth/api-keys/:id` | 删除 API Key | - |

#### Web UI
| 页面 | 说明 | 验收标准 |
|---|---|---|
| 评分组件 | 评分和评论 | 可提交评分 |
| 包统计 | 下载趋势 | 显示下载量图表 |
| API Key 管理 | 生成/删除 | CI/CD Token 管理 |
| 包编辑 | 编辑描述 | 包作者可编辑 |

#### Agent 支持
| Agent | 配置路径 | 验收标准 |
|---|---|---|
| Cursor | `~/.cursor/mcp.json` | 安装后可正常使用 |

### 验收标准
- [ ] 评分系统可用
- [ ] CI/CD Token 认证可用
- [ ] Web UI 功能完整
- [ ] 至少对接一个 OAuth Provider

### 时间
3 周

---

## Phase 3: 团队协作 (Week 7-9)

### 目标
支持团队使用，私有化部署场景。

### 用户故事
- US-008: 更新包
- US-019: 创建团队
- US-020: 管理团队成员

### 新增功能

#### 团队管理
| 功能 | 说明 | 验收标准 |
|---|---|---|
| 创建团队 | 设置名称和 slug | 团队创建成功 |
| 邀请成员 | 通过用户名 | 成员加入团队 |
| 移除成员 | 管理员操作 | 成员被移除 |
| 角色管理 | owner/admin/member | 权限正确 |
| 团队 scope | @team-name | 包名自动关联 |

#### 包可见性
| 可见性 | 说明 | 验收标准 |
|---|---|---|
| public | 所有人可见 | 搜索可找到 |
| team | 仅团队成员 | 非成员不可见 |
| private | 仅作者 | 仅自己可见 |

#### Web UI
| 页面 | 说明 | 验收标准 |
|---|---|---|
| 团队列表 | 我的团队 | 显示团队列表 |
| 团队详情 | 成员管理 | 邀请/移除成员 |
| 团队设置 | 基本信息 | 编辑团队信息 |
| 成员管理 | 角色设置 | 修改成员角色 |

### 依赖管理 (基础)
| 功能 | 说明 | 验收标准 |
|---|---|---|
| 声明依赖 | akit.json 中声明 | 格式正确 |
| 安装时检查 | 检查依赖是否存在 | 缺失时报错 |

### 验收标准
- [ ] 团队创建和管理可用
- [ ] 团队包仅成员可见
- [ ] 成员角色和权限正确
- [ ] 依赖声明和检查可用

### 时间
3 周
