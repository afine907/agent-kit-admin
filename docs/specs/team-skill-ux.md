# 团队技能管理 — 体验设计与验收标准

> 面向用户：用得爽；面向开发者：可验收

---

## 一、核心体验原则

### 1.1 零认知负担

**问题：** 设计师、产品、客服、运营不知道什么是 MCP、Skill、scope、Registry

**解决方案：**
- 界面全部用「工具包」代替「包/package」
- 「发布工具包」代替「publish package」
- 安装结果用自然语言展示：「已装好，可以在 Claude Code 里用了」
- 错误提示说人话：「这个名字已经被占了，换一个试试」

### 1.2 操作链路短

**问题：** 完成一件事需要的步骤太多，用户会中途放弃

**解决原则：**

| 操作 | 最长链路 | 说明 |
|---|---|---|
| 安装工具包（Web） | 2步 | 列表页 → 点安装 → 完成 |
| 安装工具包（CLI） | 1步 | `akit install @team/pkg` |
| 发布工具包（Web） | 2步 | 填表+选团队（1页） → 点发布（1步） |
| 发布工具包（CLI） | 1步 | `akit publish ./ --team @slug` |

**超长链路优化要求：**

| 操作 | 原步数 | 优化后 | 优化方案 |
|---|---|---|---|
| Web 发布工具包 | 5步 | **2步** | 填表+选团队合并为1页；上传后直接预览+1步发布 |
| 成员找包再安装 | 4-5步 | **2步** | 首页直接展示「我的团队工具包」入口；搜索结果页直接点安装 |
| 设计师首次安装 | 5步 | **2步** | 从团队包列表页直接点安装，不需要进入详情页 |
| CLI 更新所有包 | 1步 | **1步** | `akit update --all --team @slug` 本身已达标 |

### 1.3 反馈及时明确

每个操作完成都要有明确反馈：

```
✅ 发布成功！版本 v0.1.0，已同步到团队 @frontend
   安装命令：akit install @frontend/web-search-mcp

❌ 发布失败：包名 @frontend/web-search-mcp 已存在
   你可以：更新已有版本，或者换一个名字
```

### 1.4 本地状态可见

成员随时知道：

- 我装了哪些工具包？什么版本？
- 哪些工具有更新？
- 我的版本和团队最新版差多少？

## 三、验收标准（可测试）

### 3.1 发布验收

#### AC-01：发布工具包到团队（Web）

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 未登录用户访问发布页 | 跳转登录页 |
| 2 | 登录后进入发布页 | 看到「发布到」选择：个人 / 团队 |
| 3 | 选择「团队」 | 下拉框显示我所属的团队列表 |
| 4 | 选择 @frontend，填写包信息 | 正常填写 |
| 5 | 点击发布 | 返回 201，包出现在团队包列表 |
| 6 | 再次发布同名包 | 返回 409，提示包名已存在 |

#### AC-02：发布工具包到团队（CLI）

```bash
akit publish ./ --team @frontend

# 预期输出：
# ✅ 发布成功！
#    工具包: @frontend/web-search-mcp
#    版本: v0.1.0
#    安装: akit install @frontend/web-search-mcp
```

#### AC-03：CLI 发布校验

| 场景 | 操作 | 预期结果 |
|---|---|---|
| 目录没有 akit.json | `akit publish ./` | 报错：当前目录不是工具包项目 |
| 包名已在团队存在 | `akit publish ./ --team @frontend` | 报错：@frontend/web-search-mcp 已存在 |
| 团队不存在 | `akit publish ./ --team @nonexist` | 报错：团队 @nonexist 不存在 |

## 四、错误提示文案

### 4.1 错误文案原则

1. **说清楚是什么错**：不是「Error 500」，是「团队 @test 不存在」
2. **说清楚怎么修**：不是「Permission denied」，是「需要是团队成员才能安装，请联系 @admin」
3. **不说技术术语**：不是「unauthorized」，是「请先登录」

### 4.2 错误提示表

| 场景 | 文案 |
|---|---|
| 包名已存在 | 「@team/pkg 已存在，你可以更新已有版本，或换一个名字」 |
| 团队不存在 | 「团队 @xxx 不存在，请检查名字是否正确」 |
| 没有权限安装 | 「你没有权限安装这个工具包，请联系团队管理员」 |
| 包不存在 | 「找不到 @team/pkg，请检查包名是否正确」 |
| 安装目录不对 | 「当前目录不是工具包项目，请先进入项目目录或运行 akit init」 |
| 版本号格式不对 | 「版本号格式不对，请使用 semver 格式，如 1.0.0」 |
| 未登录 | 「请先登录：akit login --server https://xxx」 |
| Token 过期 | 「登录已过期，请重新登录：akit logout && akit login」 |

## 六、数据指标（运营参考）

### 6.1 团队维度

| 指标 | 说明 |
|---|---|
| 团队包总数 | 该团队发布的包数量 |
| 总安装次数 | 该团队所有包被安装的次数 |
| 活跃成员数 | 30天内登录过的成员数 |
| 发布者占比 | 发布过包的成员 / 总成员 |

### 6.2 平台维度

| 指标 | 说明 |
|---|---|
| 团队总数 | 所有团队数 |
| 包总数 | 所有团队 + 个人包数量 |
| 平台总安装次数 | 所有包的安装次数之和 |
| 活跃团队数 | 7天内有人操作的团队 |
