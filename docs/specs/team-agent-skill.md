# 团队技能管理 — Agent 代操作 Skill 设计

## 核心定位

> **Agent = 用户的数字同事，代替用户完成重复性包管理工作**

akit Skill 是「你让我做，我来做」，这份设计是「你不用管，我盯着做」。

区别：

| 模式 | 触发方式 | 典型场景 |
|---|---|---|
| **触发式**（当前 akit Skill） | 用户说话 | "帮我安装 @team/web-search" |
| **主动式**（本文） | Agent 自主感知 | "有新版本，我来帮你更新" |

---

## 一、Agent 能代替用户做什么

### 1.1 推荐代替的操作

| 操作 | 代替的价值 | 风险 |
|---|---|---|
| 检查团队包更新 | 不用每天手动刷 | 低（成员确认后执行） |
| 安装团队新包 | 省去 2 步操作 | 低（成员确认后执行） |
| 通知团队成员有新工具 | 省去群发消息 | 极低（只是通知） |
| 汇总团队工具使用情况 | 减少重复造轮子 | 极低（只是展示） |
| **更新包（用户确认后）** | 发现新版本，Agent 建议，成员一点确认就更新 | 低（用户确认是安全阀） |
| **发布包（用户确认后）** | 准备好发布内容，用户一点确认就发 | 低（用户确认是安全阀） |

### 1.2 不推荐代替的操作

| 操作 | 原因 |
|---|---|
| 未确认直接自动更新 | 可能破坏现有工作流，风险高 |
| 未确认直接自动发布 | 发布不可逆，且可能是误触发 |
| 自动删除包 | 不可逆，风险太高 |
| 修改包内容 | 版权和责任问题 |

> **原则：Agent 做「感知 + 通知 + 执行」，成员只需要「确认」**

## 二、Skill 文件设计

### 2.1 文件位置

```
~/.claude/skills/akit-agent.md
```

### 2.2 Skill 元数据

```markdown
---
name: akit-agent
description: Agent Kit Admin - 主动管理团队工具包
trigger: auto  # 表示持续运行，不等待用户触发
---

# akit-agent - Agent 代操作 Skill
```

### 2.3 主动监控能力

```markdown
## 能力 1：检查团队包更新

### 触发条件
- 每次用户登录 Claude Code 时
- 每天首次使用时（如果持续运行）

### 执行逻辑
1. 调用 `akit list --team @slug --check-updates`
2. 如果有更新，显示通知并附上更新内容摘要
3. 用户说「更新」→ Agent 执行 `akit update @team/pkg`
4. 展示结果 + 告知需要重启 Agent

### 输出示例
```
📦 团队 @frontend 有 1 个工具包可以更新：

🔔 db-toolkit: v1.5.0 → v1.6.0
   数据库查询性能优化，修复了连接池问题

输入「更新 db-toolkit」我来帮你更新。
```

---
用户确认后执行：

```
用户: 更新 db-toolkit
Agent: ⚡ 正在更新 db-toolkit...

✅ 更新成功！
   v1.5.0 → v1.6.0
   配置已更新: ~/.claude/mcp.json

⚠️  需要重启 Claude Code：/restart
```

## 能力 2：发现团队新工具

### 触发条件
- 用户登录时
- 用户说「团队有什么新工具」

### 执行逻辑
1. 调用 `akit list --team @frontend --recent`
2. 展示最近 7 天发布的包
3. 询问是否安装

### 输出示例
```
🆕 团队 @frontend 最近新增的工具包：

📦 @frontend/llm-summary  v0.1.0  (今天)
   长文本摘要工具，支持 10 万字
   安装次数：3

📦 @frontend/img-gen  v0.2.0  (3天前)
   图片生成工具，支持 Stable Diffusion API
   安装次数：7

要安装哪个？
```

## 能力 3：工具推荐

### 触发条件
- 用户说「我需要 xxx 相关的工具」「有没有 xxx」
- 用户问「做 xxx 用什么工具好」

### 执行逻辑
1. 调用 `akit search <keyword> --team @slug`
2. 展示搜索结果 + 安装命令
3. 询问是否安装

### 输出示例
```
🔍 搜索「客服」相关工具...

在 @frontend 找到了：

💬 客服话术生成  v0.5.0
   根据问题生成回复话术
   安装 15 次  ⭐ 4.8

💬 投诉处理助手  v0.3.0
   辅助处理投诉流程
   安装 8 次

是否安装？输入「安装 客服话术生成」
```

## 能力 4：团队工具统计

### 触发条件
- 用户问「大家都在用什么」「团队工具使用情况」

### 执行逻辑
1. 调用 `akit list --team @frontend --stats`
2. 展示安装排行、最近更新等

### 输出示例
```
📊 团队 @frontend 工具包统计：

安装排行：
1. @frontend/web-search  (23次安装)
2. @frontend/db-toolkit  (18次安装)
3. @frontend/auth-skill  (12次安装)

最近更新：
- @frontend/img-gen  v0.2.0 (3天前)
- @frontend/llm-summary  v0.1.0 (今天)

你的安装情况：
- 已安装 4 个
- 有 1 个可以更新
```

## 能力 5：安装确认

### 触发条件
- 用户说「帮我装 xxx」「安装 xxx」

### 执行逻辑
1. 执行 `akit install @team/xxx`
2. 成功后展示结果和后续步骤

### 输出示例
```
✅ 安装成功！

📦 @frontend/web-search-mcp  v1.2.0
   配置已写入: ~/.claude/mcp.json

⚠️  需要重启 Claude Code 才能生效：
   /restart

重启后即可使用网页搜索功能。
```

## 能力 6：发布辅助

### 触发条件
- 用户在项目目录说「发布这个 MCP」「发布当前工具包」

### 执行逻辑
1. 检测当前目录是否有 `akit.json`
2. 检查是否已登录
3. 显示发布预览，确认后执行

### 输出示例
```
检测到工具包项目：

📦 @frontend/my-tool  v0.1.0
   类型: MCP
   描述: 我的工具

确认发布到团队 @frontend？
输入「确认」或「取消」
```

## 三、CLI 命令扩展

为了支持 Agent 的主动能力，需要扩展 CLI：

```bash
# 检查更新的包（Agent 主动通知用）
akit list --team @slug --check-updates --json

# 最近发布的包（Agent 发现新工具用）
akit list --team @slug --recent --days 7

# 团队统计（Agent 统计用）
akit team stats @slug

# 我的安装情况（Agent 感知用户状态用）
akit list --me --team @slug --json
```

## 四、工作模式

### 4.1 被动模式（默认）

用户说话 → Agent 执行 → 等待下一步指令

```
用户: 帮我装 @frontend/web-search
Agent: ✅ 安装成功！
```

### 4.2 主动模式（需要用户授权）

用户开启主动模式 → Agent 持续监控 → 发现情况主动通知

```
# 用户开启主动模式
用户: 开启团队工具监控
Agent: ✅ 已开启！我会主动提醒你：
   - 团队有新工具时通知你
   - 有包可以更新时通知你
   - 每天汇总团队工具使用情况

# Agent 主动通知（下次使用时）
Agent: 📢 团队 @frontend 有 1 个新工具：
   @frontend/llm-summary v0.1.0 - 长文本摘要工具
   要安装吗？
```

### 4.3 开启方式

```bash
# 在 Claude Code 中
用户: 开启团队工具监控
Agent: ✅ 已开启！请告诉我你的团队 slug：

用户: @frontend
Agent: ✅ 已绑定团队 @frontend！
   你可以在任何时候问我：
   - 「团队有什么新工具」
   - 「帮我更新所有包」
   - 「搜索 xxx 相关的工具」
```

## 七、验收标准

### AC-13：Agent 推荐新工具

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 团队发布了新包 | 其他成员下次使用 Claude Code 时收到通知 |
| 2 | 用户问「团队有什么新工具」 | Agent 展示最近 7 天的新包 |
| 3 | 用户说「帮我装 xxx」 | Agent 执行安装，告知后续步骤 |

### AC-14：Agent 更新提醒

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 已安装包有新版本 | Agent 提醒有更新，询问是否更新 |
| 2 | 用户说「帮我更新」 | Agent 执行 `akit update` |
| 3 | 用户说「不用管」 | Agent 记录偏好，不再频繁提醒 |

### AC-15：团队工具统计

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 用户问「大家都在用什么」 | Agent 展示安装排行榜 |
| 2 | 用户问「我装了哪些」 | Agent 展示我的安装列表 + 状态 |
