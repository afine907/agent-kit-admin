# 用户故事

## 用户画像

### Persona 1: 小明 - MCP 开发者

- **角色**: 前端开发，业余开发 MCP 工具
- **目标**: 让团队使用自己开发的 MCP
- **痛点**: 手动分享 MCP 配置麻烦，版本更新后要通知所有人
- **场景**: 开发了一个 web-search MCP，想发布给团队使用

### Persona 2: 小红 - AI 工程师

- **角色**: AI 应用开发，重度使用 Claude Code
- **目标**: 快速找到高质量的 MCP/Skill
- **痛点**: 不知道团队有哪些可用的 MCP，配置繁琐
- **场景**: 需要一个数据库查询 MCP，想直接安装使用

### Persona 3: 老王 - 团队负责人

- **角色**: 技术负责人，管理 10 人团队
- **目标**: 统一管理团队的 AI 工具链
- **痛点**: 每个人的 Agent 配置不一样，难以协作
- **场景**: 想建立团队私有的 MCP/Skill 市场

---

## EPIC 1: 包发布

### US-001: 发布 MCP

**作为** MCP 开发者
**我想** 把我的 MCP 发布到私有 Registry
**以便** 团队其他人可以直接安装使用

**验收标准:**
- [ ] 执行 `akit publish ./` 成功发布
- [ ] 自动读取 `akit.json` 作为 manifest
- [ ] manifest 格式自动校验，错误时给出明确提示
- [ ] 版本号遵循 semver，重复版本发布报 409 错误
- [ ] 发布成功后返回包的访问地址
- [ ] 包文件存储到 MinIO
- [ ] 版本信息记录到数据库

**优先级:** P0
**Story Points:** 5

---

### US-002: 发布 Skill

**作为** Skill 开发者
**我想** 把我的 Skill 发布到私有 Registry
**以便** 团队其他人可以使用我的 Agent 技能

**验收标准:**
- [ ] 支持 `type: "skill"` 的 manifest
- [ ] Skill 内容正确存储和展示
- [ ] 安装后正确写入 Agent 配置

**优先级:** P0
**Story Points:** 3

---

### US-003: 初始化项目配置

**作为** 开发者
**我想** 通过交互式引导创建 `akit.json`
**以便** 快速开始发布我的第一个 MCP/Skill

**验收标准:**
- [ ] 执行 `akit init` 启动交互式引导
- [ ] 询问包名、版本、类型、描述等基本信息
- [ ] 生成标准格式的 `akit.json`
- [ ] 如果已存在，询问是否覆盖

**优先级:** P1
**Story Points:** 3

---

## EPIC 2: 包安装

### US-004: 安装 MCP 到 Claude Code

**作为** Claude Code 用户
**我想** 一键安装团队发布的 MCP
**以便** 无需手动编辑 `~/.claude/mcp.json`

**验收标准:**
- [ ] 执行 `akit install @team/xxx` 一条命令完成
- [ ] 自动检测已安装的 Agent
- [ ] 自动写入 `~/.claude/mcp.json` 配置
- [ ] 如果配置已存在，备份后覆盖
- [ ] 安装成功后提示需要重启 Agent
- [ ] 安装失败时有明确错误提示

**优先级:** P0
**Story Points:** 5

---

### US-005: 安装 MCP 到 Codex

**作为** Codex 用户
**我想** 一键安装团队发布的 MCP
**以便** 无需手动配置

**验收标准:**
- [ ] 自动检测 Codex 安装
- [ ] 自动写入 Codex 配置文件
- [ ] 配置格式符合 Codex 规范

**优先级:** P0
**Story Points:** 3

---

### US-006: 查看已安装列表

**作为** 开发者
**我想** 查看我已安装的 MCP/Skill 列表和版本
**以便** 管理我的本地技能

**验收标准:**
- [ ] 执行 `akit list` 展示所有已安装包
- [ ] 显示包名、当前版本、类型
- [ ] 按类型分组展示 (MCP / Skill)
- [ ] 如果有新版本可用，显示更新提示

**优先级:** P1
**Story Points:** 2

---

### US-007: 卸载包

**作为** 开发者
**我想** 卸载不再需要的 MCP/Skill
**以便** 保持环境整洁

**验收标准:**
- [ ] 执行 `akit uninstall @team/xxx` 卸载
- [ ] 删除本地包文件
- [ ] 清理 Agent 配置中的对应条目
- [ ] 确认提示 (可选 `--yes` 跳过)

**优先级:** P1
**Story Points:** 3

---

### US-008: 更新包

**作为** 开发者
**我想** 更新已安装的包到最新版本
**以便** 获取最新功能和修复

**验收标准:**
- [ ] 执行 `akit update @team/xxx` 更新单个包
- [ ] 执行 `akit update --all` 更新所有包
- [ ] 更新时备份旧版本配置
- [ ] 更新失败时自动回滚

**优先级:** P2
**Story Points:** 3
