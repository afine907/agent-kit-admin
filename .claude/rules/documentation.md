# 文档规范

## 文档类型

### 1. 项目文档 (README.md)
- 项目简介和目的
- 快速开始指南
- 安装说明
- 基本使用示例
- 贡献指南链接

### 2. API 文档
- 使用 OpenAPI/Swagger 规范
- 每个端点包含:
  - 描述
  - 请求参数
  - 请求体示例
  - 响应格式
  - 错误码说明
  - 认证要求

### 3. 架构文档
- 系统架构图 (Mermaid)
- 数据流说明
- 组件职责描述
- 技术选型理由

### 4. 开发文档
- 开发环境搭建
- 代码规范说明
- 测试指南
- 部署流程

## 编写原则

### 清晰性
- 使用简单直接的语言
- 避免歧义和模糊表述
- 提供具体示例

### 完整性
- 覆盖所有重要功能
- 包含边界情况说明
- 提供故障排除指南

### 及时性
- 代码变更时同步更新文档
- 定期审查文档准确性
- 删除过时内容

## Markdown 规范

### 标题层级
- 一级标题: 文档标题（每个文件只有一个）
- 二级标题: 主要章节
- 三级标题: 子章节
- 避免跳过标题层级

### 代码块
```markdown
# 行内代码
使用 `code` 标记变量名、文件名、命令

# 代码块
​```typescript
// TypeScript 代码示例
const greeting: string = "Hello"
​```

# 带输出的代码块
​```bash
$ npm install
added 123 packages
​```
```

### 链接和引用
- 使用有意义的链接文本
- 相对路径链接到项目内文件
- 外部链接添加说明

### 表格
```markdown
| 列1 | 列2 | 列三 |
|---|---|---|
| 内容1 | 内容2 | 内容3 |
```

## 中文文档规范

### 标点符号
- 使用中文标点：，。！？；：""''（）
- 数字和英文使用半角字符
- 中英文之间添加空格

### 排版
- 段落之间空一行
- 列表项之间不空行
- 使用中文数字（一、二、三）或阿拉伯数字根据上下文

### 示例
```markdown
## 快速开始

### 1. 安装依赖

运行以下命令安装项目依赖：

​```bash
npm install
​```

### 2. 配置环境

复制 `.env.example` 文件并重命名为 `.env`，然后修改配置项：

- `DATABASE_URL` - 数据库连接地址
- `API_KEY` - API 密钥
```
