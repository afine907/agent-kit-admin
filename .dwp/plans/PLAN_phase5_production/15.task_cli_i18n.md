# Task 15: CLI i18n support

## 1. Context

Web has i18n (i18next, zh/en). CLI has none.

## 2. Goal

CLI internationalization matching Web's approach.

## 3. Instructions

1. Add `i18next` (Node.js) to CLI dependencies
2. Create `apps/cli/src/i18n/locales/zh.json`, `en.json`
3. Support `--lang` flag and `AKIT_LANG` env var
4. Default to system locale, fallback to Chinese
5. Extract all user-facing strings to locale files
6. At minimum: error messages, success messages, help text

## 4. Acceptance Criteria

- `akit --lang en info @scope/pkg` shows English
- `akit --lang zh info @scope/pkg` shows Chinese
- Default follows system locale
- All user-facing strings externalized
- Tests pass in both languages

## 5. Validation

```bash
cd apps/cli && pnpm test && pnpm lint && pnpm typecheck
```
