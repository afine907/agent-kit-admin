# Task 6: Add web OAuth callback handler

## Context

The Login page redirects to `api.getOAuthUrl(provider)` which goes to the server's OAuth endpoint. After OAuth completes, the server redirects back but there's no `/auth/callback` route in the SPA to receive the token.

## Read Before Starting

- `apps/web/src/App.tsx` — route definitions
- `apps/web/src/pages/Login.tsx` — OAuth redirect logic
- `apps/web/src/stores/auth.ts` — auth state management
- `apps/web/src/lib/api.ts` — API client

## Goal

Add a `/auth/callback` route that extracts the JWT token from the OAuth redirect URL and stores it in auth state.

## Instructions

1. Create `apps/web/src/pages/AuthCallback.tsx`:
   - On mount, extract `token` and `refreshToken` from URL search params (or hash fragment)
   - Store them in the auth Zustand store
   - Redirect to `/` (or the `returnTo` param if present)
   - Handle errors (missing token, invalid token) with a user-friendly message
2. Add route `/auth/callback` to `App.tsx`
3. Ensure the OAuth flow works: Login → server OAuth → callback → token stored → redirected to home

## Acceptance Criteria

- [ ] `/auth/callback` route exists in App.tsx
- [ ] AuthCallback page extracts token from URL params
- [ ] Token is stored in Zustand auth store
- [ ] User is redirected to home after successful callback
- [ ] Error state is handled (missing/invalid token)
- [ ] `pnpm --filter agent-kit-web lint` passes
- [ ] `pnpm typecheck` passes
- [ ] `pnpm --filter agent-kit-web test` passes
