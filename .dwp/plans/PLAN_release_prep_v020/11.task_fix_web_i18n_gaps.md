# Task 11: Fix web i18n gaps

## Context

Several components have hardcoded Chinese or English strings instead of using i18n `t()` calls:
- `TeamPackagesTab.tsx`: hardcoded Chinese strings (加载失败, 重试, 团队还没有发布任何包, 下载, 更新, 已安装, 安装中, 安装, 删除包)
- `PackageEdit.tsx`: hardcoded English labels (Edit Package, Description, Tags, Visibility, Save Changes, etc.)

## Read Before Starting

- `apps/web/src/components/TeamPackagesTab.tsx` — hardcoded Chinese
- `apps/web/src/components/PackageEdit.tsx` — hardcoded English
- `apps/web/src/i18n/locales/zh.json` — Chinese locale
- `apps/web/src/i18n/locales/en.json` — English locale

## Goal

Replace all hardcoded strings with i18n `t()` calls and add missing keys to locale files.

## Instructions

1. In `TeamPackagesTab.tsx`, replace all hardcoded Chinese strings with `t('key')` calls
2. In `PackageEdit.tsx`, replace all hardcoded English labels with `t('key')` calls
3. Add missing translation keys to both `zh.json` and `en.json` locale files
4. Use existing naming conventions from the locale files (e.g., `teamPackages.*`, `packageEdit.*`)

## Acceptance Criteria

- [ ] No hardcoded user-facing strings in TeamPackagesTab.tsx
- [ ] No hardcoded user-facing strings in PackageEdit.tsx
- [ ] All new keys exist in zh.json and en.json
- [ ] i18n works correctly in both languages
- [ ] `pnpm --filter agent-kit-web lint` passes
- [ ] `pnpm typecheck` passes
- [ ] `pnpm --filter agent-kit-web test` passes
