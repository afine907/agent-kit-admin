# Task 5: Fix server API Key flush + download recording

## Context

Two server issues:
1. `api_key.py`'s `flush_pending_updates()` is defined but never called — `last_used_at` is never persisted
2. Download background tasks don't pass `ip_address` and `user_agent` to the Download model — those fields are always NULL

## Read Before Starting

- `apps/server/app/services/api_key.py` — flush_pending_updates at line 151
- `apps/server/app/api/packages.py` — download background task at lines 263-295
- `apps/server/app/main.py` — app lifecycle events

## Goal

Persist API Key `last_used_at` and record download metadata (ip, user_agent).

## Instructions

1. **API Key flush:** Add a shutdown event in `main.py` that calls `flush_pending_updates()` on app shutdown. Alternatively, use a periodic background task (e.g., every 60 seconds) to flush pending updates.
2. **Download recording:** In the download background task in `packages.py`, accept and pass `ip_address` and `user_agent` parameters to the Download model. Extract these from the original request in the endpoint that triggers the background task.

## Acceptance Criteria

- [ ] `flush_pending_updates()` is called periodically or on shutdown
- [ ] Download records include `ip_address` and `user_agent`
