# Task 7: Add web token refresh

## Context

The auth store has `refreshToken` fields and the API client has `api.refreshToken()`, but there's no automatic refresh logic. When JWT expires, the user gets a 401 and is logged out immediately.

## Read Before Starting

- `apps/web/src/lib/api.ts` — API client, existing 401 interceptor
- `apps/web/src/stores/auth.ts` — auth store with refreshToken

## Goal

Implement automatic token refresh: on 401, attempt refresh with the refresh token, retry the original request, and only logout if refresh fails.

## Instructions

1. In `apps/web/src/lib/api.ts`, modify the 401 response interceptor:
   - Check if a refresh token exists in the auth store
   - If yes, call `api.refreshToken()` to get a new access token
   - Update the auth store with the new token
   - Retry the original request with the new token
   - If refresh fails (also 401), then clear auth and redirect to login
2. Use a flag/queue to prevent multiple simultaneous refresh attempts (if multiple requests 401 at the same time, only refresh once)

## Acceptance Criteria

- [ ] 401 response triggers token refresh attempt
- [ ] Original request is retried with new token after successful refresh
- [ ] Multiple simultaneous 401s don't trigger multiple refreshes
- [ ] Failed refresh results in logout + redirect to login
