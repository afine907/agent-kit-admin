# Task 3: Add server leave-team endpoint

## Context

The `remove_member` endpoint at `teams.py:151` tells users "Cannot remove yourself, use leave team instead", but no `POST /api/v1/teams/{id}/leave` endpoint exists. Team members who want to self-remove are stuck.

## Read Before Starting

- `apps/server/app/api/teams.py` — existing team endpoints, especially `remove_member`
- `apps/server/app/services/teams.py` — team service logic
- `apps/server/app/models/team.py` — TeamMember model

## Goal

Add a `POST /api/v1/teams/{id}/leave` endpoint that allows a team member to remove themselves, with a guard preventing the owner from leaving.

## Instructions

1. Add `leave_team(team_id: str, user: User)` to `apps/server/app/services/teams.py`:
   - Find the user's membership record
   - If user is the owner, raise an error ("Owner cannot leave team. Transfer ownership or delete the team.")
   - Delete the membership record
2. Add `POST /api/v1/teams/{id}/leave` to `apps/server/app/api/teams.py`:
   - Require authentication
   - Call `leave_team()` service
   - Return 204 on success
3. Add a test for the leave endpoint in the test suite

## Acceptance Criteria

- [ ] `POST /api/v1/teams/{id}/leave` endpoint exists
- [ ] Owner cannot leave (returns error)
- [ ] Non-owner members can leave successfully
- [ ] Membership record is deleted
- [ ] `ruff check .` passes
- [ ] `ruff format --check .` passes
- [ ] Test passes
