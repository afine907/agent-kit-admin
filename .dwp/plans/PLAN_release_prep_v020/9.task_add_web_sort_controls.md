# Task 9: Add web sort controls

## Context

The Home page lists packages but has no sort UI. The API supports `sort` and `order` params but the frontend doesn't expose them.

## Read Before Starting

- `apps/web/src/pages/Home.tsx` — package list page
- `apps/web/src/lib/api.ts` — listPackages accepts sort/order params
- `apps/web/src/components/SearchBar.tsx` — existing filter UI

## Goal

Add sort controls to the package list page so users can sort by downloads, created date, updated date, or name.

## Instructions

1. Add a sort dropdown/select component near the search bar or filter area in `Home.tsx`
2. Options: "最新" (newest), "最热" (most downloads), "名称" (name A-Z)
3. Wire the selection to `api.listPackages({ sort, order })` params
4. Reset to page 1 when sort changes
5. Persist sort preference in URL query params for shareability

## Acceptance Criteria

- [ ] Sort dropdown is visible on Home page
- [ ] Sorting works for downloads, created date, name
- [ ] Page resets to 1 on sort change
- [ ] Sort state is reflected in URL query params
