# Task 10: Complete web PackageEdit

## Context

The PackageEdit form only allows editing description, tags, and visibility. The package model also has license, repository, and homepage fields that cannot be edited through the UI.

## Read Before Starting

- `apps/web/src/components/PackageEdit.tsx` — current edit form
- `apps/web/src/pages/PackageEdit.tsx` — page wrapper
- `apps/web/src/lib/api.ts` — updatePackage API call
- `apps/server/app/api/packages.py` — PATCH endpoint accepts these fields

## Goal

Add license, repository, and homepage fields to the PackageEdit form.

## Instructions

1. Add input fields for `license` (text input, e.g. "MIT"), `repository` (URL input), and `homepage` (URL input) to `PackageEdit.tsx`
2. Include these fields in the form state and the `api.updatePackage()` call
3. Add basic URL validation for repository and homepage fields
4. Ensure the form pre-populates with existing values when loading

## Acceptance Criteria

- [ ] License, repository, homepage fields are in the edit form
- [ ] Fields are pre-populated with current values
- [ ] Form submits all fields to the API
- [ ] URL validation for repository and homepage
- [ ] `pnpm --filter agent-kit-web lint` passes
- [ ] `pnpm typecheck` passes
- [ ] `pnpm --filter agent-kit-web test` passes
