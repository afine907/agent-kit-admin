# Task 2: Fix CLI update — re-download and apply

## Context

The `akit update` command only updates the version string in config but never downloads new files or reads the actual manifest. Agent config is rewritten with hardcoded `node index.js` values instead of the package's actual manifest values.

## Read Before Starting

- `apps/cli/src/commands/update.ts` — current incomplete implementation
- `apps/cli/src/commands/install.ts` — reference for download + extract flow (after Task 1)
- `apps/cli/src/utils/tarball.ts` — extractTarball (after Task 1)
- `apps/cli/src/utils/manifest.ts` — readManifest

## Goal

Make `akit update` actually download the new version, extract it, read the manifest, and rewrite agent config with correct values.

## Instructions

1. In `apps/cli/src/commands/update.ts`:
   - After checking the latest version, download the tarball (same as install flow)
   - Extract the tarball using `extractTarball()` (from Task 1)
   - Read the manifest from the extracted files
   - Rewrite agent config using actual manifest values (command, args, env) instead of hardcoded values
   - Update the version in configManager's installed_packages
   - Clean up the tarball after extraction
2. Ensure the update flow handles: already up-to-date, version comparison, error cases
