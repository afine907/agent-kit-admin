# Task 6: Server Tag API

## Context

- Package.tags: JSONB array of strings (`["mcp", "web"]`)
- PackageResponse schema already includes `tags: list[str]`
- No dedicated tag listing endpoint exists

## Goal

Add tag listing endpoint and tag-based filtering.

## Instructions

### 1. Add tag listing endpoint

File: `apps/server/app/api/packages.py`

```
GET /packages/tags
Response: [{"tag": "mcp", "count": 12}, {"tag": "web", "count": 7}]
```

Implementation: query all packages, flatten tags array, count occurrences using PostgreSQL JSONB array unnesting.

### 2. Enhance tag filtering

`GET /packages?tag=mcp` — filter packages where `tags` JSONB array contains the given tag.

### 3. Add schema

```python
class TagInfo(BaseModel):
    tag: str
    count: int

class TagListResponse(BaseModel):
    tags: list[TagInfo]
```

### 4. Write tests

File: `apps/server/tests/test_tag_api.py`

## Acceptance Criteria

- [ ] `GET /packages/tags` returns all tags with correct counts
- [ ] `GET /packages?tag=mcp` returns only packages with "mcp" in tags
- [ ] Tag filter combines with other filters
- [ ] Empty tag returns empty list (not error)

## Validation

```bash
cd apps/server && pytest tests/test_tag_api.py -v
cd apps/server && ruff check app/
```

## Outputs

- `apps/server/app/api/packages.py` — new endpoint + filter
- `apps/server/app/schemas/package.py` — new schemas
- `apps/server/tests/test_tag_api.py` — tests
