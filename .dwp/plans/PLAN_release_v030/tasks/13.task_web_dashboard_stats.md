# Task 13: Web Dashboard Stats

## Context

- No dashboard page exists
- Admin stats endpoint may not exist yet
- Charts: use CSS bars (no recharts dependency needed)

## Goal

Create `/dashboard` page with metrics cards, activity feed, and installs-over-time chart.

## Instructions

### 1. Add route

File: `apps/web/src/App.tsx`

```tsx
<Route path="/dashboard" element={<DashboardPage />} />
```

Link in Header navbar (admin-only).

### 2. Backend: stats endpoint

If not already implemented, add:

File: `apps/server/app/api/admin.py` (or extend existing)

```
GET /admin/stats
Response: {
  total_packages: int,
  total_installs_30d: int,
  active_teams: int,
  active_users: int,
  recent_activity: [{type: "publish"|"install", actor: str, package: str, timestamp: str}],
  installs_by_day: [{date: "2026-07-01", count: 42}]
}
```

Add to main.py router if new file.

### 3. Dashboard page

File: `apps/web/src/pages/Dashboard.tsx`

**Metrics cards (4):**
- Total Packages (count)
- Installs (last 30d)
- Active Teams
- Active Users

Use shadcn/ui `Card` component for each.

**Installs over time (bar chart):**
- Show last 14 days
- CSS-based bars: each day = a bar, height proportional to count
- X-axis: date labels
- Y-axis: count

**Recent activity feed:**
- List of recent actions (publish/install)
- Each item: icon (📦 publish, ⬇️ install) + actor + package name + relative time
- Max 10 items

### 4. Loading and error states

- Show skeleton loaders while fetching stats
- Show error message if API fails

## Acceptance Criteria

- [ ] `/dashboard` renders 4 metric cards with data
- [ ] Bar chart shows 14 days of install data
- [ ] Activity feed shows recent actions
- [ ] Skeleton loaders shown during fetch
- [ ] Error state shown on API failure

## Validation

```bash
cd apps/web && pnpm exec tsc --noEmit

```

- `apps/web/src/pages/Dashboard.tsx` — dashboard page
- `apps/server/app/api/admin.py` — stats endpoint (if missing)
- `apps/web/src/App.tsx` — route added
