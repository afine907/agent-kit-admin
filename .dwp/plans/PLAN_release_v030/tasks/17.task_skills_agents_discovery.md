# Task 17: Skills & Agents Discovery

## Context

- Skills catalog: `.agents/skills/` directory
- Agents catalog: `.agents/agents/` directory

## Goal

Review v0.3.0 work for new reusable patterns worth capturing as skills or agents.

## What to Check

### Patterns from v0.3.0 that might deserve a skill:

1. **Team RBAC pattern** — `apps/server/app/services/team.py` role checking logic
   → Is this generic enough to be a "rbac-service" skill?

2. **Dependency resolution algorithm** — `apps/cli/src/utils/dependency-resolver.ts` cycle detection + topological sort
   → Is this worth a "dependency-resolver" skill?

3. **Invite token flow** — generation + expiry + acceptance
   → Could be a "invite-flow" skill

4. **React tabbed management page** — Teams.tsx with tabs
   → Could be a "tabbed-management-ui" pattern

5. **API token management UI** — Settings page token list/create/revoke
   → Could be a "token-management-ui" pattern

### Also check:

- Do any existing skills need updating?
- Are there agent archetypes in `docs/` that should be extracted as agents?
- Does the new `dependency-resolver.ts` logic have enough reuse potential?

## Deliverable

File: `analysis_results/SKILLS_AGENTS_DISCOVERY.md`

Structure:
- Summary: N patterns reviewed, N skills recommended, N agents recommended
- Pattern #N: name, description, evidence (files/functions), recommendation (new skill / update existing / no-op), rationale
- Agent candidates (if any)

## Acceptance Criteria

- [ ] All 5 pattern areas evaluated
- [ ] Recommendations are actionable (new skill / update / skip)
- [ ] If new skills recommended, SKILL.md content drafted

## Validation

```bash
ls .agents/skills/   # review existing catalog
ls .agents/agents/   # review existing agents
```

## Outputs

- `analysis_results/SKILLS_AGENTS_DISCOVERY.md` — discovery report
- Any new skill proposal files (SKILL.md) if recommended
