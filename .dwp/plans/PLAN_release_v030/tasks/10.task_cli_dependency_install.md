# Task 10: CLI Dependency Install

## Context

- Current `akit install` does flat install, no dependency resolution
- Server Task 8 adds `GET /packages/{scope}/{name}/dependencies` endpoint
- Install command: `apps/cli/src/commands/install.ts`

## Goal

Enhance `akit install` to support recursive dependency resolution with cycle detection.

## Instructions

### 1. Add dependency resolver utility

File: `apps/cli/src/utils/dependency-resolver.ts`

```typescript
interface DepNode {
  scope: string;
  name: string;
  version: string;
  dependencies: DepNode[];
}

interface ResolvedDeps {
  nodes: DepNode[];
  cycles: string[][];
}

// DFS with cycle detection and recursion depth limit (10)
async function resolveDependencies(
  scope: string,
  name: string,
  visited: Set<string> = new Set(),
  stack: string[] = [],
  depth: number = 0
): Promise<{ node: DepNode; cycles: string[][] }>
```

### 2. Add dry-run flag

File: `apps/cli/src/commands/install.ts`

Add `--dry-run` flag:

```bash
akit install @team/pkg --dry-run
# Output:
# Package: @team/pkg@1.0.0
# Dependencies:
#   @team/dep-a@1.0.0
#     @team/dep-b@2.0.0
#   Cycle detected: @team/a → @team/b → @team/a (skipped)
# Total: 2 packages to install
```

### 3. Enhance install command

- On `akit install @team/pkg`, first call dependency endpoint
- Build full tree, detect cycles
- Show installation plan (or dry-run output)
- Install in topological order (skip cycled packages with warning)
- Store dependency info in install record

### 4. Write tests

File: `apps/cli/tests/utils/dependency-resolver.test.ts`

Mock API responses for: normal deps, deep deps (5+ levels), cycle detection.

## Acceptance Criteria

- [ ] `akit install @team/pkg --dry-run` shows full dependency tree
- [ ] `akit install @team/pkg` installs package and all dependencies
- [ ] Cycle detected → warning printed, cycled packages skipped
- [ ] Deep dependencies (>10 levels) → truncated with warning
- [ ] Install order respects topological sort
