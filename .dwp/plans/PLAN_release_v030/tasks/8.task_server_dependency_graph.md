# Task 8: Server Dependency Graph

## Context

- `POST /packages/check-dependencies` already exists
- Package model has no stored `manifest_dependencies` yet
- Dependency check endpoint currently does flat resolve only

## Goal

Implement full dependency graph: recursive resolution, cycle detection, manifest parsing.

## Instructions

### 1. Add manifest_dependencies to Package model

File: `apps/server/app/models/package.py`

```python
manifest_dependencies: Mapped[dict | None] = mapped_column(CompatJSONB, nullable=True)  # {"lodash": "^4.17.0", "express": "^5.0.0"}
```

### 2. Add migration

```bash
cd apps/server && alembic revision --autogenerate -m "add manifest_dependencies to packages"
```

### 3. Parse manifest on publish

File: `apps/server/app/api/packages.py` (publish route)

When a tarball is uploaded, extract `manifest.json` or `package.json` from the tarball and store the `dependencies` dict in `manifest_dependencies`.

### 4. Enhance dependency check endpoint

File: `apps/server/app/services/dependency.py` (or enhance in packages.py)

```python
async def resolve_dependency_graph(package_name: str, version: str | None = None) -> dict:
    """
    Recursively resolve dependency graph.
    Returns:
      {
        "dependencies": {"pkg": {"version": "1.0.0", "deps": {...}}},
        "cycles": [["a", "b", "a"]]  # list of cycles if any
      }
    """
```

Algorithm:
- Use DFS with a recursion stack
- If a node is already in the current stack → cycle detected, return the cycle path
- Limit recursion depth to 10 to prevent stack overflow

### 5. Add GET endpoint

```
GET /packages/{scope}/{name}/dependencies
→ returns {dependencies: {...}, cycles: [...]}
```

### 6. Write tests

File: `apps/server/tests/test_dependency.py`

Test: normal resolution, deep dependency tree, cycle detection (A→B→C→A).

## Acceptance Criteria

- [ ] Package publish parses and stores manifest dependencies
- [ ] `GET /packages/{scope}/{name}/dependencies` returns full tree
- [ ] Cycle detected and reported (e.g., A depends on B, B depends on A)
- [ ] Recursion depth limited to 10 levels
- [ ] Version compatibility checked (semver)
