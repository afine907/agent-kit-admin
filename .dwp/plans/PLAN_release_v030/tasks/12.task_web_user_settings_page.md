# Task 12: Web User Settings Page

## Context

- No settings page exists yet
- API client: `apps/web/src/lib/api.ts`
- Auth store: `apps/web/src/stores/authStore.ts`

## Goal

Create `/settings` page with Profile, Security, and (UI-only) Notifications sections.

## Instructions

### 1. Add route

File: `apps/web/src/App.tsx`

```tsx
<Route path="/settings" element={<SettingsPage />} />
```

Add nav link in Header user dropdown: "Settings".

### 2. Settings page layout

Three sections using shadcn/ui `Tabs`:

```tsx
<Tabs defaultValue="profile">
  <TabsList>
    <TabsTrigger value="profile">Profile</TabsTrigger>
    <TabsTrigger value="security">Security</TabsTrigger>
    <TabsTrigger value="notifications">Notifications</TabsTrigger>
  </TabsList>
  ...
</Tabs>
```

### 3. Profile tab

- Display name input (text)
- Avatar URL input (URL)
- Save button → PUT /users/me

### 4. Security tab

**Change password:**
- Current password input
- New password input + confirm input
- Save → POST /users/me/change-password (new endpoint, implement if missing)

**API Tokens:**
- List: GET /users/me/tokens
  - Show: name, created_at, expires_at, last_used (or "Never")
- Create token: POST /users/me/tokens body: {name, expires_in_days}
  - Show token ONCE after creation (modal with copy button)
- Revoke token: DELETE /users/me/tokens/{id}

### 5. Notifications tab (UI only)

- Toggle: "Email on new package publish"
- Toggle: "Email on team member joined"
- Label: "Coming soon" note

### 6. API endpoints (backend if not exists)

If endpoints don't exist, create them:

```
GET /users/me → {id, email, name, avatar_url, ...}
PUT /users/me → update name, avatar_url
POST /users/me/tokens → create API token
GET /users/me/tokens → list tokens
DELETE /users/me/tokens/{id} → revoke token
POST /users/me/change-password (server task)
```

## Acceptance Criteria

- [ ] `/settings` route loads with 3 tabs
- [ ] Profile save works
- [ ] API token list renders with create/revoke
- [ ] New token shown once in modal with copy button
- [ ] Notifications toggles render with "Coming soon" label
- [ ] Active nav highlight in Header for Settings link
