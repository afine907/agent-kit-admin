# Task 11: Web Team Management Page

## Context

- Current `Teams.tsx` is read-only
- Design system: shadcn/ui, Tailwind CSS, zustand stores
- API client: `apps/web/src/lib/api.ts`

## Goal

Rewrite `Teams.tsx` into a full team management UI with tabs: Overview, Members, Packages, Settings.

## Instructions

### 1. Add tabbed layout

Use shadcn/ui `Tabs` component:

```tsx
<Tabs defaultValue="overview">
  <TabsList>
    <TabsTrigger value="overview">Overview</TabsTrigger>
    <TabsTrigger value="members">Members</TabsTrigger>
    <TabsTrigger value="packages">Packages</TabsTrigger>
    <TabsTrigger value="settings">Settings</TabsTrigger>
  </TabsList>
  <TabsContent value="overview">...</TabsContent>
  <TabsContent value="members">...</TabsContent>
  <TabsContent value="packages">...</TabsContent>
  <TabsContent value="settings">...</TabsContent>
</Tabs>
```

### 2. Overview tab

- Team name, description, avatar (fetched from GET /teams/{team_id}/settings)
- Stats cards: Member count, Package count, Total installs
- Recent activity: last 5 publishes/installs

### 3. Members tab

- Table: Avatar | Name | Email | Role badge | Joined | Actions
- Role badges: owner=purple, admin=blue, member=gray
- Actions: role dropdown (owner only), remove button (owner only, with confirm dialog)
- Invite button → opens modal with copy invite link

### 4. Packages tab

- Grid of PackageCard components (reuse from Home.tsx)
- Filter by visibility (all/team/public)

### 5. Settings tab

- Form: Team name, description, avatar URL, default visibility dropdown
- Save button (admin+ only); non-admin sees read-only view with message

### 6. Invite modal

- Trigger: "Invite Member" button
- Content: "Your invite link:" + copyable link
- Copy-to-clipboard button

### 7. API calls

```typescript
GET /teams?name=<team> → team_id
GET /teams/{team_id}/settings
GET /teams/{team_id}/members
PUT /teams/{team_id}/members/{user_id}/role
DELETE /teams/{team_id}/members/{user_id}
POST /teams/{team_id}/invites
GET /teams/{team_id}/packages
PUT /teams/{team_id}/settings
```

## Acceptance Criteria

- [ ] All 4 tabs render and are navigable
- [ ] Member list shows roles with correct badge colors
- [ ] Role change works from dropdown (owner only)
- [ ] Remove member shows confirmation dialog
- [ ] Invite modal shows copyable link
- [ ] Settings tab saves and shows updated values
- [ ] Non-admin cannot access Settings tab content
