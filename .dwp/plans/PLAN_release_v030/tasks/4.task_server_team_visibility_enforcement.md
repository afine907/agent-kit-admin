# Task 4: Server Team Visibility Enforcement

## Context

- `Package.visibility`: `public` | `private` | `team`
- team visibility = only TeamMember can see/install/download
- `is_team_member()` helper exists in PackageService
- Some endpoints may already filter; need full audit

## Goal

Audit and fix all endpoints to enforce team visibility correctly. Non-members get 404 (not 403) to avoid leaking existence.

## Instructions

### 1. Audit all package endpoints

Check each of these returns 404 for team-packages when caller is not a member:

- `GET /packages` — list; team packages filtered out for non-members
- `GET /packages/{scope}/{name}` — detail; 404 if team and not member
- `GET /packages/{scope}/{name}/stats` — 404 if team and not member
- `GET /packages/{scope}/{name}/download` — 404 if team and not member
- `GET /packages/{scope}/{name}/versions/{version}/download` — 404 if team and not member
- `GET /packages/{scope}/{name}/versions` — 404 if team and not member
- `GET /packages/{scope}/{name}/dependencies` — 404 if team and not member
- `POST /packages/check-dependencies` — should process but not expose private data
- `GET /teams/{team_id}/packages` — ensure only members can list

### 2. Create visibility helper

File: `apps/server/app/services/package.py` (or new `visibility.py`)

```python
async def check_package_visibility(pkg: Package, user_id: str | None) -> bool:
    """Return True if user can access package. False → return 404."""
    if pkg.visibility == "public":
        return True
    if pkg.visibility == "private":
        return pkg.owner_id == user_id
    if pkg.visibility == "team":
        return await is_team_member(pkg.scope, user_id)  # or scope as team_id
    return False
```

### 3. Apply to all endpoints

Add inline visibility checks or use a decorator `@require_package_access`.

### 4. Write tests

File: `apps/server/tests/test_visibility.py`

- Test team package → non-member gets 404 at each endpoint
- Test team package → member gets 200
- Test public/private unchanged behavior

## Acceptance Criteria

- [ ] All package read endpoints return 404 for team-packages to non-members
- [ ] All package download endpoints return 404 for team-packages to non-members
- [ ] Team members can access team packages normally
- [ ] Public and private packages behave unchanged
