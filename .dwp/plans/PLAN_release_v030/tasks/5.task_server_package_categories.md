# Task 5: Server Package Categories

## Context

- Package model in `apps/server/app/models/package.py`
- Version model in `apps/server/app/models/version.py`
- PackageResponse schema in `apps/server/app/schemas/package.py`

## Goal

Add `category` field to Package model, expose in API, support filtering.

## Instructions

### 1. Add category field to Package model

File: `apps/server/app/models/package.py`

```python
category: Mapped[str | None] = mapped_column(String(50), nullable=True, index=True)
```

### 2. Add migration

```bash
cd apps/server && alembic revision --autogenerate -m "add category to packages"
```

### 3. Update schemas

File: `apps/server/app/schemas/package.py`

- `PackageResponse` → add `category: str | None`
- `PackageCreate` → add `category: str | None = None`
- `PackageUpdate` → add `category: str | None = None`

### 4. Update API

File: `apps/server/app/api/packages.py`

- `GET /packages` → support `?category=foo` filter query param
- `PUT /packages/{scope}/{name}` → support updating `category` field

### 5. Write tests

File: `apps/server/tests/test_package_categories.py`

## Acceptance Criteria

- [ ] PackageCreate accepts `category`
- [ ] PackageResponse includes `category`
- [ ] `GET /packages?category=foo` returns only matching packages
- [ ] `PUT /packages/{scope}/{name}` with `{"category": "mcp"}` updates the field
- [ ] Category filter works alongside existing filters (visibility, tag, search)

## Validation

```bash
cd apps/server && pytest tests/test_package_categories.py -v
cd apps/server && ruff check app/
cd apps/server && mypy app/
```
