# Task 15: E2E Verification

## Context

- pytest-asyncio for server tests
- Existing journey tests in `apps/server/tests/test_e2e_journey.py`

## Goal

End-to-end test covering the complete v0.3.0 team workflow.

## Instructions

File: `apps/server/tests/test_e2e_v030.py`

Cover these scenarios:

### Scenario 1: Complete team lifecycle
```python
1. Create team_A (user1)
2. Generate invite → user2 joins via token
3. Verify user2 is member
4. user1 promotes user2 to admin
5. Verify user2 can invite others
```

### Scenario 2: Package publish + install under team
```python
1. Setup: team + 2 members
2. Publish @team/testpkg as member
3. Install @team/testpkg as member2
4. Verify install record created
```

### Scenario 3: Role management
```python
1. owner → admin (promote)
2. owner → member (transfer → accept)
3. Verify old owner now member
```

### Scenario 4: Visibility enforcement
```python
1. Publish team-only package
2. Log in as outsider → GET /packages → package NOT in list
3. Log in as team member → GET /packages → package visible
```

### Scenario 5: Deprecation warning
```python
1. Publish package
2. Deprecate with reason
3. GET /packages/{pkg} → deprecated=true, deprecation_reason set
```

### Scenario 6: Dashboard stats
```python
1. Publish 2 packages
2. Install 5 times
3. GET /admin/stats → total_packages >= 2, total_installs_30d >= 5
```

## Acceptance Criteria

- [ ] All 6 scenarios pass
- [ ] Tests use async with pytest-asyncio
- [ ] Each scenario is independent (can run in any order)
- [ ] Proper setup/teardown for each test

## Validation

```bash
cd apps/server && pytest tests/test_e2e_v030.py -v
```

## Outputs

- `apps/server/tests/test_e2e_v030.py` — comprehensive E2E tests
