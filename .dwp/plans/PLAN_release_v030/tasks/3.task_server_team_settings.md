# Task 3: Server Team Settings

## Context

- Team model has: name, slug, description
- Team settings will include: avatar_url, default_visibility, and the above

## Goal

Add team settings CRUD: GET/PUT team settings, admin+ only.

## Instructions

### 1. Add TeamSettings model

File: `apps/server/app/models/team.py`

```python
class TeamSettings(Base):
    __tablename__ = "team_settings"

    team_id: Mapped[uuid.UUID] = mapped_column(CompatUUID, ForeignKey("teams.id"), primary_key=True)
    avatar_url: Mapped[str | None] = mapped_column(String(500), nullable=True)
    default_visibility: Mapped[str] = mapped_column(String(20), default="team")  # public|private|team
    website: Mapped[str | None] = mapped_column(String(500), nullable=True)
    updated_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

### 2. Add service methods in TeamService

- `get_settings(team_id)` — return TeamSettings or create defaults
- `update_settings(team_id, user_id, updates)` — validate admin+, update fields

### 3. Add API routes

File: `apps/server/app/api/teams.py`

- `GET /teams/{team_id}/settings` — return settings (member+)
- `PUT /teams/{team_id}/settings` — update settings (admin+); body: avatar_url, default_visibility, website

### 4. Add schemas

File: `apps/server/app/schemas/team.py`

```python
class TeamSettingsResponse(BaseModel): avatar_url: str|None; default_visibility: str; website: str|None
class TeamSettingsUpdate(BaseModel): avatar_url: str|None=None; default_visibility: str|None=None; website: str|None=None
```

### 5. Add migration

```bash
cd apps/server && alembic revision --autogenerate -m "add team_settings table"
```

## Acceptance Criteria

- [ ] GET returns full settings (or defaults if none)
- [ ] PUT updates only provided fields
- [ ] Non-admin gets 403 on PUT
- [ ] Member can GET settings
- [ ] default_visibility must be one of public/private/team
