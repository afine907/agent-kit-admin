# Task 3: Package ownership transfer

## 1. Context

No ownership transfer endpoint exists. Package owners need to transfer packages to other users or teams.

**Files to modify:**
- `apps/server/app/api/packages.py` — add transfer endpoint
- `apps/server/app/services/package.py` — add transfer logic
- `apps/server/tests/test_transfer.py` — new test file

## 2. Goal

Package owners can transfer ownership to another user or team via API.

## 3. Instructions

1. Add endpoint: `POST /api/v1/packages/{scope}/{name}/transfer`
2. Request body: `{"new_owner": "@username"}` or `{"new_owner": "@team-slug"}`
3. Logic:
   - Only current owner can transfer (403 otherwise)
   - Target user/team must exist (404 otherwise)
   - Update `packages.owner_id` and `packages.scope`
   - Log the transfer (structured log)
4. After transfer, new owner can publish versions, old owner cannot
5. Write tests: successful transfer, permission check, invalid target, self-transfer blocked

## 4. Acceptance Criteria

- Owner transfers → package shows new owner
- Non-owner → 403
- Target not found → 404
- New owner can publish after transfer
- Old owner cannot publish after transfer
- Tests cover all scenarios (≥5 test cases)
