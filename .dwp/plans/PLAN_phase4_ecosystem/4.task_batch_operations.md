# Task 4: Batch operations

## 1. Context

No batch operation endpoints exist. Users need to deprecate or delete multiple packages at once.

**Files to modify:**
- `apps/server/app/api/packages.py` — add batch endpoints
- `apps/server/app/services/package.py` — add batch logic
- `apps/server/tests/test_batch.py` — new test file

## 2. Goal

Batch delete and batch deprecate endpoints for packages.

## 3. Instructions

1. `POST /api/v1/packages/batch/delete` — body: `{"packages": ["@scope/name1", "@scope/name2"]}`
2. `POST /api/v1/packages/batch/deprecate` — body: `{"packages": [...], "deprecated": true}`
3. Only owner/team-admin can batch operate on their packages
4. Soft delete (set `deleted_at`)
5. Return: `{"success": [...], "failed": [{"name": "...", "error": "..."}]}`
6. Max 50 packages per batch
7. Write tests: batch delete, batch deprecate, permission per package, max limit

## 4. Acceptance Criteria

- Batch delete soft-deletes multiple packages
- Batch deprecate marks/unmarks multiple packages
- Unauthorized packages → error in `failed` array (not whole request fails)
- Max 50 limit → 400 error
- Tests pass (≥6 test cases)
