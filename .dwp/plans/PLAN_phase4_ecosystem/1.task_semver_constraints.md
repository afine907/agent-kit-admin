# Task 1: Semver constraint support in dependency resolver

## 1. Context

The dependency resolver (`apps/server/app/services/dependency.py`) currently only checks if a package name exists. It does NOT validate version constraints (e.g., `^1.0.0`, `~2.1.0`).

**Files to modify:**
- `apps/server/app/services/dependency.py` — add semver logic
- `apps/server/app/schemas/package.py` — update dependency schema
- `apps/server/app/api/packages.py` — update checkDependencies endpoint
- `apps/server/tests/test_dependency.py` — add constraint tests
- `apps/server/pyproject.toml` — add `semver` dependency

## 2. Goal

Packages can declare dependencies with version constraints in `akit.json`:
```json
{"dependencies": {"@scope/pkg": "^1.0.0"}}
```
The system validates that an available version satisfies the constraint.

## 3. Instructions

1. Add `semver` Python package to `pyproject.toml`
2. Extend `DependencyResolver`:
   - New method `check_constraint(version: str, constraint: str) -> bool`
   - Support: exact (`1.0.0`), caret (`^1.0.0`), tilde (`~1.0.0`), wildcard (`1.x`, `1.*`)
3. Update `checkDependencies` API endpoint:
   - Accept `dependencies: {pkg_name: {"name": "@scope/pkg", "constraint": "^1.0.0"}}`
   - For each dependency, find the latest version satisfying the constraint
   - Return error if no version satisfies constraint
4. Write tests: exact match, caret range, tilde range, wildcard, no match, invalid constraint format

## 4. Acceptance Criteria

- `dependencies: {"@scope/pkg": "^1.0.0"}` resolves when matching version exists
- Returns error when no version satisfies constraint
- Invalid constraint format returns clear error
- Existing dependency existence check still works (backward compatible)
- Unit tests cover all semver range types (≥6 test cases)
