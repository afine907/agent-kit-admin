# Task 2: Webhook notification system

## 1. Context

No webhook system exists. Teams need to receive notifications when packages are published/deleted.

**Files to create:**
- `apps/server/app/models/webhook.py` — SQLAlchemy model
- `apps/server/app/services/webhook.py` — business logic
- `apps/server/app/api/webhooks.py` — API endpoints
- `apps/server/app/tasks/webhook_delivery.py` — async delivery
- `apps/server/tests/test_webhook.py` — tests

## 2. Goal

Teams can register webhooks to receive HTTP POST notifications on package events.

## 3. Instructions

1. **Model** — `webhooks` table: id (UUID PK), team_id (FK), url (TEXT), secret (VARCHAR), events (JSONB array), active (BOOLEAN), created_at, updated_at
2. **API endpoints:**
   - `POST /api/v1/teams/{team_id}/webhooks` — create webhook (owner/admin only)
   - `GET /api/v1/teams/{team_id}/webhooks` — list webhooks
   - `DELETE /api/v1/teams/{team_id}/webhooks/{id}` — delete webhook
3. **Events:** `package.published`, `package.deleted`, `version.published`, `version.yanked`
4. **Delivery:** async background task, HMAC-SHA256 signature, retry 3x exponential backoff
5. **Payload:** `{event, timestamp, package: {name, scope, version}, actor: {username}}`
6. **Integration:** fire webhooks from package publish/delete/version endpoints
7. Write tests: CRUD + delivery + signature verification

## 4. Acceptance Criteria

- Team owner/admin can CRUD webhooks
- Webhook fires on package publish with correct payload
- HMAC-SHA256 signature is valid
- Failed deliveries retry up to 3 times
- Non-admin cannot create webhooks (403)
- Unit + integration tests pass (≥8 test cases)
