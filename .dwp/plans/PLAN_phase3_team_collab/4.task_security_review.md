# Task 4: Security Review

## 1. Context

This is the mandatory security review task (third-to-last). Reviews the plan's accumulated diff for security issues.

## 2. Goal

Audit all changes made in Tasks 1-3 for security concerns and verify `docs/SECURITY.md` is current.

## 3. Instructions

1. Run `git diff master..feature/phase3-team-collab` to review all changes
2. Check for:
   - Hardcoded secrets or credentials in the diff
   - Injection risks in new CLI input handling
   - Unsafe input handling in deprecated/yanked checks
   - New attack surface from visibility filtering changes
   - Auth/permission changes that could be bypassed
3. Audit any new dependencies introduced
4. Verify `docs/SECURITY.md` still reflects reality
5. Write findings in `analysis_results/SECURITY_REVIEW.md`

## 4. Acceptance Criteria

- `analysis_results/SECURITY_REVIEW.md` written (even if "no findings")
- No hardcoded secrets in any commit
- `docs/SECURITY.md` updated if needed
- Critical findings fixed before plan completion

## 5. Validation

```bash
# Check for secrets in diff
git diff master..HEAD -- '*.ts' '*.py' | grep -iE "(password|secret|token|api_key)" || echo "No secrets found"
# Verify security doc exists
cat docs/SECURITY.md | head -5
```
