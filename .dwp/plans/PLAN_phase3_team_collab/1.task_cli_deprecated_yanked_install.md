# Task 1: CLI install — warn/block on deprecated and yanked versions

## 1. Context

The CLI `install` command (`apps/cli/src/commands/install.ts`) downloads and installs packages but does not check if the resolved version is deprecated or yanked. The server already exposes these fields in version responses.

**Files to modify:**
- `apps/cli/src/commands/install.ts` — add deprecated/yanked checks
- `apps/cli/tests/commands/install.test.ts` — add test cases

## 2. Goal

When a user runs `akit install @scope/pkg`:
- If version is **deprecated** → show yellow warning, allow install
- If version is **yanked** → show red error, exit code 1
- `--force` flag bypasses yanked block

## 3. Instructions

> **Before starting:** Re-read the plan README section 1 (Goal) to ensure this task's work aligns with the overall objective.

1. Read `apps/cli/src/commands/install.ts` to understand current flow
2. After version resolution, check `version.deprecated` and `version.yanked` from API response
3. Add logic:
   - `deprecated === true` → `console.warn(chalk.yellow('⚠️ 版本 X.Y.Z 已废弃，建议升级到最新版本'))` — continue install
   - `yanked === true` → `console.error(chalk.red('❌ 版本 X.Y.Z 已撤回，无法安装。请使用其他版本。'))` → `process.exit(1)`
   - If `--force` flag present and yanked → show warning instead of error, continue
4. Add `--force` option to the install command if not present
5. Write unit tests covering: deprecated install (passes), yanked install (blocks), yanked+force (passes with warning)

## 4. Acceptance Criteria

- `akit install @scope/pkg` where version is deprecated → shows warning, installs successfully
- `akit install @scope/pkg` where version is yanked → shows error, exits code 1
- `akit install @scope/pkg --force` where version is yanked → shows warning, installs
- All existing install tests still pass
- New unit tests cover all three scenarios
