# Task 2: CLI agent adapter 测试

## 1. Context

- 影响范围：`apps/cli/src/agents/` 目录下的 adapter 实现
- 测试框架：Vitest
- Agent adapter 是 CLI 的核心架构，负责将包配置写入不同 AI agent 的配置文件

## 2. Goal

为 Claude Code adapter 和 Codex adapter 编写全面的单元测试，验证 detect/readConfig/writeConfig/removeConfig 四个核心操作。

## 3. Instructions

> **Before starting:** Re-read the plan README section 1 (Goal) to ensure this task's work aligns with the overall objective.

### 3.1 Claude Code adapter 测试

- detect: 检测 `~/.claude/mcp.json` 是否存在
- readConfig: 读取并解析 JSON 配置
- writeConfig: 写入 MCP server 配置（transport, command, args, env）
- removeConfig: 移除指定包的配置
- 测试配置文件不存在时的创建逻辑
- 测试 JSON 格式错误时的处理

### 3.2 Codex adapter 测试

- detect: 检测 `~/.codex/config.toml` 是否存在
- readConfig: 读取并解析 TOML 配置
- writeConfig: 写入 TOML 格式的配置
- removeConfig: 移除指定包的配置
- 测试配置文件不存在时的创建逻辑

### 3.3 adapter registry 测试

- 测试自动发现所有已注册的 adapter
- 测试根据 agent 类型选择正确的 adapter
- 测试未知 agent 类型的错误处理
