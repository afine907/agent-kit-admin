## 3. Instructions

> **Before starting:** Re-read the plan README section 1 (Goal) to ensure this task's work aligns with the overall objective.

### 3.1 publish → install → update 流程

1. 用户注册/登录
2. 创建 scope
3. publish 包（上传 tarball + manifest）
4. install 包（下载 + 解压 + 配置写入）
5. publish 新版本
6. update 包（下载新版本 + 解压 + 配置重写）

### 3.2 团队包管理流程

1. 创建团队
2. 添加成员
3. publish 团队包
4. 成员安装团队包
5. 成员卸载团队包安装记录
6. 成员离开团队
7. 验证 owner 不能离开

### 实现方式

- 扩展 `apps/server/tests/test_e2e_journey.py`
- 使用 conftest.py 中的 fixtures
- 每个流程是一个独立的测试函数
