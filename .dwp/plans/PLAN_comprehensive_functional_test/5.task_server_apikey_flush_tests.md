# Task 5: Server API Key flush + download metadata 测试

## 1. Context

- 影响范围：`apps/server/app/main.py` (periodic flush), `apps/server/app/api/packages.py` (download metadata)
- 测试框架：pytest + pytest-asyncio
- v0.2.0 新增 API Key `last_used_at` 周期性刷新和 download 记录中的 `ip_address`/`user_agent`

## 2. Goal

为 API Key flush 机制和 download metadata 记录编写测试。

## 3. Instructions

> **Before starting:** Re-read the plan README section 1 (Goal) to ensure this task's work aligns with the overall objective.

### 3.1 API Key flush 测试

- 测试 `last_used_at` 在 API 调用后被更新
- 测试周期性 flush 任务正确持久化到数据库
- 测试 shutdown 时的 flush 行为
- 使用 mock 时间推进或直接验证数据库状态

### 3.2 download metadata 测试

- 测试 download 端点记录 `ip_address`（从 request 中提取）
- 测试 download 端点记录 `user_agent`（从 request header 中提取）
- 测试缺少 header 时的默认值处理

### 实现方式

- 扩展 `apps/server/tests/test_api_keys.py` 或新建 `test_api_key_flush.py`
- 扩展 `apps/server/tests/test_packages.py` 中的 download 测试
