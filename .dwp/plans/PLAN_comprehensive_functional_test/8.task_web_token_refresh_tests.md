# Task 8: Web token refresh 测试

## 1. Context

- 影响范围：`apps/web/src/lib/api.ts`
- 测试框架：Vitest
- v0.2.0 新增自动 token refresh 机制，使用 promise queue 防止并发 refresh

## 2. Goal

为 token refresh 逻辑编写全面的单元测试。

## 3. Instructions

> **Before starting:** Re-read the plan README section 1 (Goal) to ensure this task's work aligns with the overall objective.

### 测试用例

1. **401 触发 refresh** — API 返回 401 时自动触发 token refresh
2. **refresh 成功后重试** — refresh 成功后重试原始请求并返回结果
3. **refresh 失败登出** — refresh 失败时清除 token 并重定向到登录页
4. **并发请求只 refresh 一次** — 多个并发 401 只触发一次 refresh（promise queue）
5. **refresh token 过期** — refresh token 也过期时的处理
6. **非 401 错误不触发 refresh** — 403/500 等错误不触发 refresh

### Mock 策略

- Mock fetch/axios 验证请求行为
- Mock localStorage 验证 token 操作
- Mock window.location 或 router 验证重定向
