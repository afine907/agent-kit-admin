# Task 1: CLI install/update 单元测试

## 1. Context

- 影响范围：`apps/cli/src/utils/tarball.ts`, `apps/cli/src/commands/install.ts`, `apps/cli/src/commands/update.ts`
- 测试框架：Vitest，测试文件与源文件同目录或在 `tests/` 下
- v0.2.0 新增了 `extractTarball()` 函数和 install/update 的完整流程

## 2. Goal

为 CLI 的 install 和 update 命令编写全面的单元测试，覆盖正常路径、边界条件和错误处理。

## 3. Instructions

> **Before starting:** Re-read the plan README section 1 (Goal) to ensure this task's work aligns with the overall objective.

### 3.1 tarball.test.ts

为 `extractTarball()` 编写测试：
- 正常解压 tarball 到目标目录
- 处理损坏的 tarball 文件（应抛出有意义的错误）
- 处理空 tarball
- 路径遍历防护（tarball 内含 `../` 路径）
- strip: 1 行为验证（解压后不包含顶层目录）

### 3.2 install.test.ts

为 install 命令编写 mock 测试：
- Mock download 函数，验证 extractTarball 被调用
- Mock manifest 读取，验证配置写入逻辑
- 测试下载失败时的错误处理
- 测试解压失败时的清理逻辑（.tar.gz 文件应被删除）
- 测试 manifest 不存在时的处理

### 3.3 update.test.ts

为 update 命令编写 mock 测试：
- Mock 下载 → 解压 → manifest 读取 → 配置重写完整流程
- 测试版本不变时的行为
- 测试下载失败时的回滚
- 测试配置重写使用 manifest 中的实际值
